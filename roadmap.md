# Roadmap

## Current Phase: Content & Validation

### Core Principle
"Your wife is the first learner, not the only learner."

The Visit to Aotearoa course is **reusable semantic content**. Learner goals are **personal**. The curriculum graph is **generated per learner**.

### Architecture

```
Shared Course (semantic intents, surface forms, native audio)
      ↓
Learner Onboarding (what do you want to do? → goals)
      ↓
Personal Curriculum Graph (learn / recognize / say per concept)
      ↓
Personal Practice + Feedback
```

### Learner Goals (configurable per concept)
- **Learn** — "I want to know this concept exists"
- **Recognize** — "I want to understand when someone says it"
- **Say** — "I want to produce this myself"

### Onboarding Flow
Instead of "what language?", ask:
1. "What do you want to do with the language?" (visit family / travel / work / conversation / culture / revitalization)
2. "What do you want to be able to do?" (understand / say things / converse / read / pronounce / culture)

### Pronunciation Feedback — Three Scores
1. **Recognition** — Did the system understand you? (binary: yes/no)
2. **Similarity** — How close was the sound? (automated comparison)
3. **Human review** — Native-speaker feedback on recording (optional)

System must clearly distinguish machine feedback from native-speaker feedback.

### Immediate Target
Wife learns **20–25 genuinely useful Māori expressions** for a real family visit.
This is the proof of concept.

### Course YAML Refactor
- Separate reusable content (phrases, entities, semantic intents) from learner goals
- Learner profile becomes a separate config, not baked into the course
- Curriculum graph is generated by matching learner goals against semantic content

### Future Work (Post-Validation)
- Generic learner model extraction
- Pronunciation feedback pipeline
- Multi-course support
- Speech recognition integration
